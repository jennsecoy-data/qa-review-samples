# Edit Verification Log

This sample log is used to track requested edits, compare approved source content against the final version, document issues, and confirm whether corrections were completed.

## Purpose

An edit verification log helps ensure that requested changes are applied accurately and consistently across documents, forms, records, online content, and supporting materials.

This type of log is useful for:

- Tracking requested edits
- Comparing source content to final content
- Documenting missing or incorrect changes
- Confirming corrections after updates
- Identifying repeated issues
- Supporting clear communication with a lead, reviewer, or production team

## Edit Verification Table

| Item | Source Requirement | Final Version Check | Issue Found | Severity | Status |
|---|---|---|---|---|---|
| Sample Form A | Update instruction text to approved wording | Final text matches source | None | None | Pass |
| Sample Form B | Replace image caption with approved caption | Caption updated, but punctuation differs | Minor formatting inconsistency | Low | Needs Review |
| Sample Online Item | Add required accessibility note | Note is missing from online version | Missing required edit | Medium | Needs Correction |
| Sample Record Entry | Verify customer/account date | Date matches, but account detail differs | Data mismatch | Medium | Needs Correction |
| Sample Document Page | Confirm final heading format | Heading spacing matches other sections | None | None | Pass |
| Sample Ancillary Material | Apply same wording used in main document | Wording differs from main version | Cross-version inconsistency | Medium | Needs Review |
| Sample Text-to-Speech Version | Confirm spoken text matches final item text | One sentence is missing | Incomplete alternate-format content | High | Needs Correction |

## Review Status Guide

| Status | Meaning |
|---|---|
| Pass | Edit was applied correctly and no issue was found |
| Needs Review | Minor issue or uncertainty should be checked before approval |
| Needs Correction | Edit was missing, incorrect, incomplete, or inconsistent |
| Blocked | Review cannot continue because required information is missing |
| Complete | Correction was verified after update |

## Severity Guide

| Severity | Meaning |
|---|---|
| Low | Minor formatting or style issue that does not change meaning |
| Medium | Accuracy, consistency, or completeness issue that should be corrected |
| High | Missing, incorrect, or conflicting information that may affect final quality or usability |

## Example Notes

### Sample Form B

The requested caption was applied, but punctuation does not match the approved source. Recommend confirming whether punctuation should be standardized across all versions before final approval.

### Sample Online Item

The required accessibility note is missing from the online version. This should be corrected before approval because the paper and online materials do not currently match.

### Sample Text-to-Speech Version

One sentence from the final item text is missing from the text-to-speech version. Recommend correcting the alternate-format version and completing a second verification pass.

## Final Verification Steps

Before closing an edit verification task, confirm:

- All requested edits were reviewed
- Source and final versions were compared
- Issues were documented clearly
- Corrections were sent back for update when needed
- Updated versions were rechecked
- Final status was recorded
- Repeated issues were noted for possible process improvement

## Summary

This sample demonstrates how edit verification can support accuracy, consistency, documentation quality, issue tracking, and clear follow-up during QA review workflows.
